      14             : //============================================================================
      15             : // Note that SPHEREPACK (which is wrapped by YlmSpherepack) takes
      16             : // n_theta and n_phi as input, and is ok with arbitrary values of
      17             : // n_theta and n_phi.  SPHEREPACK computes the maximum Ylm l and m
      18             : // using the formulas l_max=n_theta-1 and m_max=min(l_max,n_phi/2).
      19             : //
      20             : // However, some combinations of n_theta and n_phi have strange properties:
      21             : // - The maximum m that is AT LEAST PARTIALLY represented by (n_theta,n_phi)
      22             : //   is std::min(n_theta-1,n_phi/2). This is called m_max here. But an arbitrary
      23             : //   (n_theta,n_phi) does not necessarily fully represent all m's up to m_max,
      24             : //   because sin(m_max phi) might be zero at all collocation points, and
      25             : //   therefore sin(m_max phi) might not be representable on the grid.
      26             : // - The largest m that is fully represented by (n_theta,n_phi) is
      27             : //   m_max_represented = std::min(n_theta-1,(n_phi-1)/2).
      28             : // - Therefore, if n_phi is odd,  m_max = m_max_represented,
      29             : //              if n_phi is even, m_max = m_max_represented+1.
      30             : // - To remedy this situation, we choose YlmSpherepack to take as arguments
      31             : //   l_max and m_max, instead of n_theta and n_phi.
      32             : //   We then choose
      33             : //      n_theta = l_max+1
      34             : //      n_phi   = 2*m_max+1
      35             : //   This ensures that m_max = m_max_represented
      36             : //   (as opposed to m_max = m_max_represented+1)
